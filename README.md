## ToDo API

軽量かつ高速なバックエンド環境を実現するための、APIファースト設計に基づいたToDo管理アプリケーションです。

## 使い方

標準の使い方（推奨）

```sh
fastapi run main.py
```

開発時（手元での動作確認）

```sh
fastapi dev main.py
```

開発機での動作確認や、将来的なRDBMS移行検証用

```sh
docker compose up
```

## 開発の背景とコンセプト

本プロジェクトは、Webサイトを構築するための「MVCフレームワーク（Rails等）」の導入ではなく、ハードウェアの計算資源を最適化した「APIサーバー」の構築を目的としています。

- APIファーストの設計: ブラウザ画面での操作だけでなく、将来的にCLIツールや自動化スクリプトから直接叩ける柔軟性を重視しました。
- 計算資源の最適化: 最終的なデプロイ先をラズパイ等のシングルボードコンピュータ（SBC）に設定しています。Railsのようなフルスタックフレームワークが消費するリソースを避け、FastAPIを採用することで、低スペックなデバイスでも軽快に動作する設計としています。
- 必要十分なセキュリティ: 個人利用というユースケースを前提とし、ログイン機能の導入によるUX低下を避け、APIキーによる最小限のアクセス管理を採用しています。

## 技術スタック

- Language: Python 3.x
- Framework: FastAPI
- Data Storage: SQLite + JSONバックアップ（ポータビリティ重視）

## 運用の考え方（トレードオフ）

本アプリは「過剰なエンジニアリングを排除する」ことを方針としています。

- DB選定: 本格的なRDBMSではなく、SQLiteとJSONバックアップを採用しています。これにより、サーバー構築時の複雑さを排除し、バックアップを「ファイルのコピー」だけで完結させています。
- ログ管理: ディスク肥大化のリスクを避けるため、冗長なログ出力は行わず、標準出力での運用を前提としています。
- 将来の拡張: 現在は最小構成ですが、データベースの接続ロジックを分離しているため、将来的にSBC（Raspberry Pi等）へのセルフホスト移行や、PostgreSQL等への本格的なRDBMS移行を行う際も、ビジネスロジック（API側）のコードを1行も書き換えることなくシームレスに移行できる疎結合設計となっています。
